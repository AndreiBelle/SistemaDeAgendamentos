1 - Empresa estava enfrentando problemas com agendamentos da sala da reunião, tudo estava sendo feito no "caderninho" e como isso causando conflitos ao utilizar a sala;

2 - Em uma conversa com a colaboradora responsável pelos agendamentos entendemos as Regras de Negócios para esse pequeno projeto;

3 - Criado uma pequena CRUD para a colaboradora realizar os primeiros agendamentos e mais duas máquinas testes para realizar verificações;

4 - Adicionado segunda sala de reunião no sistema para ser incluída nos agendamentos também, e alterado algumas validações e detalhes como por exemplo: agendamentos realizados 
para a sala do 13ª andar vão estar destacados com a cor azul fraco para melhor vizualização;

5 - Projeto atendeu as espectativas dos usuários e principalmente da colaboradora que realizava os agendamentos, ficando no momento apenas com a função de "gerenciar" os agendamentos;

6 - Pequena função de "Logs dos agendamentos" foi adicionada como um pequeno controle, então a cada criação ou edição de agendamentos ele registra com o objetivo de caso necessário
podemos verificar ações dentro do software, no nosso caso está armazenando no servidor com acesso restrito.
